## 1. 核心概念

### 1.1 镜像（Image）

镜像是一个**只读模板**，包含运行应用所需的代码、运行时、库、环境变量和配置文件。

可以用类比来理解：
- **镜像 ≈ 类的定义**（静态、不可变）
- **容器 ≈ 类的实例**（运行中、可交互）

镜像通过 `docker build` 构建，通过 `docker pull` 从仓库拉取。

### 1.2 容器（Container）

容器是镜像的**运行实例**。在镜像之上增加一个**可写层**（容器层），所有运行时修改（创建文件、修改环境变量、安装包）都发生在这个层。

```
┌──────────────────────────┐  ← 容器层（可写，运行时产生）
│  运行时修改（新增/修改）  │
├──────────────────────────┤
│  镜像层 5：应用代码       │  ← 只读
├──────────────────────────┤
│  镜像层 4：依赖库         │
├──────────────────────────┤
│  镜像层 3：系统工具       │
├──────────────────────────┤
│  镜像层 2：基础系统       │
├──────────────────────────┤
│  镜像层 1：引导文件系统   │
└──────────────────────────┘
```

### 1.3 层（Layer）与写时复制

镜像由多个**只读层**叠加而成，每个 `Dockerfile` 指令通常产生一个新层。多个镜像可共享底层，节省存储与拉取时间。

**写时复制（Copy-on-Write）**：容器需要修改某个文件时，才会从只读层复制一份到容器层，原始文件保持不变。

### 1.4 仓库（Registry）

仓库用来**存储和分发**镜像，类似代码仓库。Docker Hub 是公共仓库，企业常自建私有仓库（如 Harbor）。

镜像命名规范：`[仓库地址/][用户名/]镜像名[:标签]`

```
docker.io/library/nginx:latest        # Docker Hub 官方 nginx 最新版
langgenius/dify-api:1.16.0-rc1        # dify 自有仓库的 API 镜像
```

## 2. 代码示例

### 2.1 基本命令

```bash
# 拉取镜像
docker pull nginx:latest

# 查看本地镜像
docker images

# 从镜像启动容器
docker run -d --name my-nginx -p 8080:80 nginx:latest
# -d: 后台运行
# --name: 容器名
# -p 8080:80: 宿主机 8080 端口映射到容器 80 端口

# 查看运行中的容器
docker ps

# 进入容器终端
docker exec -it my-nginx /bin/bash

# 查看容器日志
docker logs my-nginx

# 停止/删除容器
docker stop my-nginx
docker rm my-nginx
```

### 2.2 自己构建镜像

```dockerfile
# 文件：Dockerfile
FROM python:3.12-slim          # 基础镜像（第 1 层）
WORKDIR /app                   # 切换工作目录（第 2 层）
COPY requirements.txt .        # 复制依赖文件（第 3 层）
RUN pip install -r requirements.txt   # 安装依赖（第 4 层）
COPY . .                       # 复制应用代码（第 5 层）
CMD ["python", "app.py"]       # 启动命令
```

```bash
docker build -t my-app:1.0 .
docker run -d --name my-app -p 5000:5000 my-app:1.0
```

### 2.3 常见错误：使用 `latest` 标签

```yaml
# ❌ 错误：latest 标签不可重现
services:
  api:
    image: langgenius/dify-api:latest

# ✅ 正确：使用具体版本
services:
  api:
    image: langgenius/dify-api:1.16.0-rc1
```

## 3. dify 仓库源码解读

### 3.1 dify 的 API 镜像

**文件位置**：`/Users/xu/code/github/dify/docker/docker-compose.yaml`
**核心代码**（行 227-271）：

```yaml
  # API service
  api:
    <<: *shared-api-worker-config
    image: langgenius/dify-api:1.16.0-rc1
    environment:
      MODE: api
      SENTRY_DSN: ${API_SENTRY_DSN:-}
      PLUGIN_MAX_PACKAGE_SIZE: ${PLUGIN_MAX_PACKAGE_SIZE:-52428800}
      AGENT_BACKEND_BASE_URL: ${AGENT_BACKEND_BASE_URL:-http://agent_backend:5050}
    depends_on:
      init_permissions:
        condition: service_completed_successfully
      redis:
        condition: service_started
      agent_backend:
        condition: service_started
    volumes:
      - ./volumes/app/storage:/app/api/storage
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5001/health"]
      interval: 30s
      timeout: 5s
      retries: 3
    networks:
      - ssrf_proxy_network
      - default
```

**解读**：
- 第 1 行：`<<: *shared-api-worker-config` YAML 锚点引用，复用通用配置（`env_file`/`networks`/`restart`）
- 第 2 行：`image` 字段指定具体镜像 `langgenius/dify-api:1.16.0-rc1`，**不用 latest 保证可重现部署**
- 第 9-12 行：`depends_on` 控制启动顺序，等 `redis` 起来后再启动 API
- 第 19-23 行：`healthcheck` 用 `curl` 检查 `/health` 端点，30s 间隔，最多重试 3 次
- 第 26-27 行：`networks` 把容器接入 `ssrf_proxy_network`（内网隔离）和 `default`（默认网桥）

### 3.2 镜像版本管理

**文件位置**：`/Users/xu/code/github/dify/docker/docker-compose.yaml`
**核心代码**（行 1-5）：

```yaml
# ==================================================================
# WARNING: This file is auto-generated by generate_docker_compose
# Do not modify this file directly. Instead, update the .env.example
# or docker-compose-template.yaml and regenerate this file.
# ==================================================================
```

**解读**：
- dify 用 `docker-compose-template.yaml` 作为**模板**，通过 `generate_docker_compose` 脚本生成最终文件
- 改镜像版本应改模板和 `.env.example`，**不要直接编辑生成的 yaml**
- 这是基础设施即代码的常见模式：模板 + 自动生成，避免人工编辑出错

## 4. 关键要点总结

- **镜像**是静态模板，**容器**是运行实例，**仓库**是分发中心
- 镜像采用**分层存储**，多个镜像可共享底层，节省空间
- 容器在镜像之上增加一个**可写层**（Copy-on-Write）
- 生产环境**绝不用 `latest` 标签**，必须用具体版本号
- dify 的服务定义字段：`image` + `environment` + `depends_on` + `volumes` + `networks` + `healthcheck`
