## 1. 核心概念

### 1.1 Compose 是什么？

Docker Compose 是**用 YAML 文件定义和运行多容器应用**的工具。一条命令就能启动/停止整套应用栈。

### 1.2 核心概念

- **Service（服务）**：一个容器实例的定义
- **Network（网络）**：服务间通信的虚拟网络
- **Volume（卷）**：持久化数据（独立于容器生命周期）
- **Profile（配置）**：可选服务分组（按需启动）

### 1.3 常用字段

```yaml
services:
  web:                              # 服务名（DNS 名称）
    image: nginx:1.27               # 镜像
    ports:                          # 端口映射
      - "8080:80"
    environment:                    # 环境变量
      - NGINX_HOST=example.com
    volumes:                        # 挂载
      - ./html:/usr/share/nginx/html
    depends_on:                     # 启动依赖
      - api
    networks:                       # 加入网络
      - frontend
    restart: always                 # 自动重启
    healthcheck:                    # 健康检查
      test: ["CMD", "curl", "-f", "http://localhost"]

networks:
  frontend:

volumes:
  data:
```

### 1.4 Profile：按需启动

把可选服务放进 `profiles`，默认不启动：

```bash
docker compose --profile elasticsearch up
```

dify 用 Profile 把 PostgreSQL、MySQL、Qdrant、Milvus 等多种数据库设为可选。

## 2. 代码示例

### 2.1 简单的多容器应用

```yaml
# 文件：docker-compose.yaml
services:
  api:
    image: my-api:1.0
    environment:
      DATABASE_URL: postgres://user:pass@db:5432/mydb
    depends_on:
      - db
      - redis
    networks:
      - backend

  web:
    image: my-web:1.0
    ports:
      - "3000:3000"
    environment:
      API_URL: http://api:8000
    depends_on:
      - api
    networks:
      - frontend
      - backend

  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: pass
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - backend

  redis:
    image: redis:7-alpine
    networks:
      - backend

volumes:
  db-data:

networks:
  frontend:
  backend:
```

```bash
docker compose up -d
docker compose ps
docker compose logs -f api
docker compose down
```

### 2.2 健康检查：服务启动顺序

```yaml
services:
  api:
    image: my-api:1.0
    depends_on:
      db:
        condition: service_healthy   # 等待 db healthy 后才启动

  db:
    image: postgres:15
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "postgres"]
      interval: 5s
      timeout: 3s
      retries: 10
```

### 2.3 常见错误：环境变量硬编码

```yaml
# ❌ 错误：密码硬编码
environment:
  DB_PASSWORD: mysecretpassword

# ✅ 正确：使用 .env 文件
environment:
  DB_PASSWORD: ${DB_PASSWORD}

# .env 文件（不进入版本控制）
DB_PASSWORD=mysecretpassword
```

## 3. dify 仓库源码解读

### 3.1 dify 的服务编排骨架

**文件位置**：`/Users/xu/code/github/dify/docker/docker-compose.yaml`
**核心代码**（行 1-72）：

```yaml
# ==================================================================
# WARNING: This file is auto-generated by generate_docker_compose
# Do not modify this file directly. Instead, update the .env.example
# or docker-compose-template.yaml and regenerate this file.
# ==================================================================

# Shared configuration using YAML anchors and env_file
x-shared-api-worker-config: &shared-api-worker-config
  env_file:
    - path: ./envs/core-services/shared.env
      required: false
    - path: ./envs/core-services/api.env
      required: false
    - path: ./envs/security.env
      required: false
    - path: ./envs/databases/db-postgres.env
      required: false
    - path: ./envs/databases/db-mysql.env
      required: false
    - ./.env
  networks:
    - ssrf_proxy_network
    - default
  restart: always
```

**解读**：
- 第 8 行：`x-shared-api-worker-config` 是 YAML **本地自定义类型**（`x-` 前缀），用 `&` 锚点标记
- 第 10-23 行：合并多个 env_file 引用，`required: false` 表示文件不存在不报错
- 第 25-26 行：所有 API/Worker 服务都加入 `ssrf_proxy_network`（隔离内网）和 `default`
- 第 27 行：`restart: always` 容器退出后自动重启
- **YAML 锚点精髓**：`&shared-api-worker-config` 声明，`<<: *shared-api-worker-config` 引用，避免配置重复

### 3.2 Profile：可选的数据库和向量库

**文件位置**：`/Users/xu/code/github/dify/docker/docker-compose.yaml`
**核心代码**（行 416-451）：

```yaml
  # The PostgreSQL database.
  db_postgres:
    image: postgres:15-alpine
    profiles:
      - postgresql
    restart: always
    environment:
      POSTGRES_USER: ${DB_USERNAME:-postgres}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-difyai123456}
      POSTGRES_DB: ${DB_DATABASE:-dify}
      PGDATA: ${PGDATA:-/var/lib/postgresql/data/pgdata}
    command: >
      postgres -c 'max_connections=${POSTGRES_MAX_CONNECTIONS:-100}'
               -c 'shared_buffers=${POSTGRES_SHARED_BUFFERS:-128MB}'
               -c 'work_mem=${POSTGRES_WORK_MEM:-4MB}'
               -c 'maintenance_work_mem=${POSTGRES_MAINTENANCE_WORK_MEM:-64MB}'
    volumes:
      - ./volumes/db/data:/var/lib/postgresql/data
    healthcheck:
      test:
        [
          "CMD",
          "pg_isready",
          "-h",
          "db_postgres",
          "-U",
          "${DB_USERNAME:-postgres}",
          "-d",
          "${DB_DATABASE:-dify}",
        ]
      interval: 1s
      timeout: 3s
      retries: 60
```

**解读**：
- 第 4-5 行：`profiles: [postgresql]` 把 PostgreSQL 设为可选，默认不启动
- 第 9-11 行：环境变量用 `${VAR:-默认值}` 语法，未设置时用默认值
- 第 13-18 行：`command: >` 用 YAML 折叠语法把多行 PostgreSQL 参数合并为一行命令
- 第 19-20 行：`./volumes/db/data` 主机目录挂载到容器 `/var/lib/postgresql/data` 实现持久化
- 第 21-30 行：健康检查用 `pg_isready`，1s 间隔检测，最多 60 次重试

## 4. 关键要点总结

- Compose 用 YAML 描述**完整应用栈**（多服务 + 网络 + 卷）
- **YAML 锚点**（`&` 声明，`*` 引用，`<<:` 合并）消除配置重复
- **Profile** 让可选服务（备选数据库/中间件）按需启动
- `${VAR:-default}` 语法给环境变量提供默认值
- `depends_on` + `healthcheck` 控制服务启动顺序
- dify 启动命令：`docker compose --profile postgresql up -d`
