## 每周节奏（所有 Phase 通用）

```text
本周阶段：Phase N
本周唯一主题：（一句话；禁止三个主题并行）
双轨时间盒（可调）：
  - ~40% Python 够用（文档 ≤2 篇 + 1 个小脚本/checkpoint）
  - ~40% Dify（跟读/改代码/画图）
  - ~20% Docker·日志·笔记·回顾
贡献（Phase 3+）：issue 链接 或 写「本周不贡献」
周末：只对照「本阶段毕业验收」打勾
未毕业 → 下周重复，不进入下一 Phase
```

## 跟读一条链路（固定 5 步）

```text
1. UI 或 curl 触发一次真实请求
2. Network / 容器日志：方法、路径、状态码
3. controllers 搜 path → service → models/core
4. 笔记：5～8 步时序 + 文件路径
5. 小改动（日志/校验/文案）并自测
```

## 范围

### 做

- Docker Compose 拓扑与排障  
- 后端 `dify/api/`  
- 登录、租户（Minimal）、应用、对话、知识库、工作流、Agent  
- 中期：小贡献 + 基础测试/日志  

### 不做 / 延后

| 项 | 处理 |
|----|------|
| `dify/web` 精读 | 当黑盒 UI |
| 全部模型 provider | 只跟 1 条 |
| 计费 billing | Phase 6 可选 |
| K8s / 多集群 | 非入门条件 |
| Python 元类/GIL/性能全书 | 用到再学 |
| `01` 后半、`02` 设计模式扫荡 | 主链路后再说 |
| 全部向量库对比 | 只用 compose 里启用的那一个 |
