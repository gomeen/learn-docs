## 上手操作

### 1. 从工厂往下点（只读）

建议打开顺序（每文件先读文件头 docstring / 主函数）：

| 序 | 文件 | 你要找到的东西 |
|----|------|----------------|
| 1 | `app.py` | 进程如何启动应用 |
| 2 | `app_factory.py` | `create_app`、扩展 `init_app` |
| 3 | `extensions/ext_blueprints.py` | 蓝图如何挂上 app |
| 4 | `controllers/console/__init__.py` | `Blueprint(..., url_prefix=...)`、`console_ns` |
| 5 | 任选一个资源，如 `controllers/console/auth/login.py` | `@console_ns.route("/login")` |

### 2. 用搜索建立肌肉记忆

在 `api/` 下：

```bash
cd /Users/xu/code/github/dify/api
rg -n "url_prefix" controllers/console/__init__.py
rg -n "console_ns.route" controllers/console/auth/login.py
rg -n "register|blueprint|console" extensions/ext_blueprints.py | head
```

### 3. 浏览器对照（需 Phase 0 环境）

1. 打开控制台，DevTools → Network  
2. 刷新或点任意会发 XHR 的操作  
3. 找一条路径以 **`/console/api/`** 开头的请求  
4. 记下：方法、完整 path、状态码  
5. 用 path 后半段在 `controllers/console` 里 `rg` 路由字符串  

示例（以你 Network 真实路径为准）：

```text
浏览器: POST /console/api/login
代码:   Blueprint url_prefix=/console/api  +  @route("/login")
文件:   controllers/console/auth/login.py
```

## 期望时序（跟读提纲）

```text
1. 容器/进程加载 app.py（或 gunicorn 指向的入口）
2. app_factory.create_app() 创建 Flask app
3. 各 extensions/*.init_app（db、login、blueprint…）
4. ext_blueprints 注册 console Blueprint
5. 请求进入：METHOD + /console/api/...
6. Flask 路由匹配到 console_ns 上的 Resource/视图
7. 视图内：鉴权装饰器（若有）→ 调 services → 返回 JSON
```

不必背每一行，但要能按此顺序在文件之间跳转。

## 关键锚点（当前仓库）

| 点 | 位置 |
|----|------|
| Console 前缀 | `controllers/console/__init__.py`：`url_prefix="/console/api"` |
| Namespace | 同文件 `console_ns` |
| 登录路由示例 | `controllers/console/auth/login.py`：`@console_ns.route("/login")` |
| 装配 | `app_factory.py` · `extensions/ext_blueprints.py` |

路径随上游变更时以本机 `rg` 结果为准。

---

## 常见卡点

| 现象 | 说明 |
|------|------|
| 找不到路由 | 只搜了函数名；应搜 **路径字符串** 或 `console_ns.route` |
| 路径对不上 | 漏了 `url_prefix`；完整 path = prefix + route |
| 和 service_api 搞混 | `controllers/service_api` 是 API Key 对外接口；控制台跟 **console** |
| 一上来读 core/ | 2.1 只要求进到 controller；业务引擎后置 |
| RESTX 看不懂 | 先当「带 Swagger 的 Resource」，细节开卡壳再读 |
