## 背景

开源协作是议题选择、沟通渠道与发布节奏的总和。验证：模拟一次从 Issue 到 PR 的路径规划。

## 需求

1. 阅读 `CONTRIBUTING.md` 与 Issue/PR 模板，写「新手第一次贡献」检查清单（10 步内）。
2. 选一个虚构/真实 good first issue，写重现步骤与范围边界（不做无关重构）。
3. `NOTES.md`：Discord/Discussions/Issue 分别适合什么问题；安全漏洞应走哪里。
4. 说明语义化版本：fix/feat/breaking 如何影响 changelog（举例 3 个提交信息）。

## 提示

- `/Users/xu/code/github/dify/CONTRIBUTING.md`
- `.github/ISSUE_TEMPLATE/`
- 禁止在公开渠道贴密钥与用户数据

## 验收标准

- [ ] 10 步清单可执行
- [ ] Issue 范围清晰、有验收
- [ ] 渠道选择表正确
- [ ] 3 个 commit 消息符合约定风格
