## 1. 核心概念

### 1.1 什么是 i18n？

i18n（Internationalization）= 让应用支持多语言（中文、英文、日文……）

Spring 通过 `MessageSource` 抽象：
- 根据 `Locale`（语言环境）加载对应的资源文件
- 资源文件命名：`messages_{lang}.properties` 或 `messages_{lang}_{country}.properties`
- 通过 `LocaleResolver` 解析当前请求的 `Locale`

### 1.2 资源文件命名规则

```
messages.properties          默认（兜底）
messages_zh.properties       中文
messages_zh_CN.properties    简体中文（中国）
messages_en.properties       英文
messages_en_US.properties    美式英语
messages_ja.properties       日文
```

**查找顺序**（精确 → 粗略）：
1. `messages_zh_CN.properties`（最精确）
2. `messages_zh.properties`
3. `messages.properties`（兜底）

### 1.3 三种 LocaleResolver

| 实现 | 解析方式 | 适用 |
|------|---------|------|
| `AcceptHeaderLocaleResolver` | HTTP `Accept-Language` 头 | API（最常用） |
| `CookieLocaleResolver` | Cookie | 传统 Web（用户偏好） |
| `SessionLocaleResolver` | Session | 传统 Web（用户登录） |

## 2. 代码示例

### 2.1 基础配置

```java
// 文件：I18nConfig.java
@Configuration
public class I18nConfig {

    @Bean
    public LocaleResolver localeResolver() {
        return new AcceptHeaderLocaleResolver();  // 基于 Accept-Language
    }

    @Bean
    public MessageSource messageSource() {
        ReloadableResourceBundleMessageSource source = new ReloadableResourceBundleMessageSource();
        source.setBasename("classpath:i18n/messages");
        source.setDefaultEncoding("UTF-8");
        source.setFallbackToSystemLocale(false);
        return source;
    }
}
```

### 2.2 资源文件

```properties
# 文件：i18n/messages_zh_CN.properties
user.not-found=用户不存在
user.create-success=用户创建成功
user.email-invalid=邮箱格式错误

# 文件：i18n/messages_en.properties
user.not-found=User not found
user.create-success=User created successfully
user.email-invalid=Email format is invalid
```

### 2.3 使用 MessageSource

```java
@Service
@RequiredArgsConstructor
public class UserService {

    private final MessageSource messageSource;

    public void validateEmail(String email) {
        if (!isValidEmail(email)) {
            String msg = messageSource.getMessage("user.email-invalid", null, LocaleContextHolder.getLocale());
            throw new ServiceException(400, msg);
        }
    }
}
```

### 2.4 在 Controller 使用

```java
@GetMapping("/hello")
public CommonResult<String> hello() {
    String msg = messageSource.getMessage("hello.world", null, LocaleContextHolder.getLocale());
    return CommonResult.success(msg);
}
```

## 3. 关键要点总结

- **i18n = 国际化**（多语言支持）
- **核心组件**：`MessageSource` + `LocaleResolver` + 资源文件
- **3 种 LocaleResolver**：`AcceptHeaderLocaleResolver`（API 推荐）、`CookieLocaleResolver`、`SessionLocaleResolver`
- **资源文件**：`messages_{lang}_{country}.properties`
- **使用方式**：`messageSource.getMessage(key, args, locale)`
- **ruoyi 设计**：错误码 + 前端 i18n，后端不依赖 `MessageSource`
