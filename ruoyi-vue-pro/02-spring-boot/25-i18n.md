## 1. 核心概念

### 1.1 什么是 i18n？

i18n（Internationalization）= 让应用支持多语言（中文、英文、日文……）

Spring 通过 `MessageSource` 抽象：
- 根据 `Locale`（语言环境）加载对应的资源文件
- 资源文件命名：`messages_{lang}.properties` 或 `messages_{lang}_{country}.properties`
- 通过 `LocaleResolver` 解析当前请求的 `Locale`

### 1.2 资源文件命名规则

```
messages.properties          默认（兜底）
messages_zh.properties       中文
messages_zh_CN.properties    简体中文（中国）
messages_en.properties       英文
messages_en_US.properties    美式英语
messages_ja.properties       日文
```

**查找顺序**（精确 → 粗略）：
1. `messages_zh_CN.properties`（最精确）
2. `messages_zh.properties`
3. `messages.properties`（兜底）

### 1.3 三种 LocaleResolver

| 实现 | 解析方式 | 适用 |
|------|---------|------|
| `AcceptHeaderLocaleResolver` | HTTP `Accept-Language` 头 | API（最常用） |
| `CookieLocaleResolver` | Cookie | 传统 Web（用户偏好） |
| `SessionLocaleResolver` | Session | 传统 Web（用户登录） |

## 2. 代码示例

### 2.1 基础配置

```java
// 文件：I18nConfig.java
@Configuration
public class I18nConfig {

    @Bean
    public LocaleResolver localeResolver() {
        return new AcceptHeaderLocaleResolver();  // 基于 Accept-Language
    }

    @Bean
    public MessageSource messageSource() {
        ReloadableResourceBundleMessageSource source = new ReloadableResourceBundleMessageSource();
        source.setBasename("classpath:i18n/messages");
        source.setDefaultEncoding("UTF-8");
        source.setFallbackToSystemLocale(false);
        return source;
    }
}
```

### 2.2 资源文件

```properties
# 文件：i18n/messages_zh_CN.properties
user.not-found=用户不存在
user.create-success=用户创建成功
user.email-invalid=邮箱格式错误

# 文件：i18n/messages_en.properties
user.not-found=User not found
user.create-success=User created successfully
user.email-invalid=Email format is invalid
```

### 2.3 使用 MessageSource

```java
@Service
@RequiredArgsConstructor
public class UserService {

    private final MessageSource messageSource;

    public void validateEmail(String email) {
        if (!isValidEmail(email)) {
            String msg = messageSource.getMessage("user.email-invalid", null, LocaleContextHolder.getLocale());
            throw new ServiceException(400, msg);
        }
    }
}
```

### 2.4 在 Controller 使用

```java
@GetMapping("/hello")
public CommonResult<String> hello() {
    String msg = messageSource.getMessage("hello.world", null, LocaleContextHolder.getLocale());
    return CommonResult.success(msg);
}
```

### 3.1 错误消息国际化（ServiceExceptionUtil）

通过 `CommonResult.error(ErrorCode, params)` 的实现可以看到 ruoyi 的 i18n 机制：

**文件位置**：`/Users/xu/code/github/ruoyi-vue-pro/yudao-framework/yudao-common/src/main/java/cn/iocoder/yudao/framework/common/pojo/CommonResult.java`
**核心代码**（行 60-66）：

```java
public static <T> CommonResult<T> error(ErrorCode errorCode, Object... params) {
    Assert.notEquals(GlobalErrorCodeConstants.SUCCESS.getCode(), errorCode.getCode(), "code 必须是错误的！");
    CommonResult<T> result = new CommonResult<>();
    result.code = errorCode.getCode();
    result.msg = ServiceExceptionUtil.doFormat(errorCode.getCode(), errorCode.getMsg(), params);
    return result;
}
```

**解读**：
- 第 5 行：`ServiceExceptionUtil.doFormat(code, msgTemplate, params)` 格式化错误消息
- **设计意图**：
  - `errorCode.getMsg()` 是模板，如 `"用户 {} 不存在，id={}"`
  - `params` 替换占位符：`("用户 1 不存在，id=1")`
  - 进一步可结合 `MessageSource` 做 i18n

### 3.2 GlobalExceptionHandler 处理异常

**文件位置**：`/Users/xu/code/github/ruoyi-vue-pro/yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/web/core/handler/GlobalExceptionHandler.java`
**核心代码**（行 130-160）：

```java
/**
 * 处理 ServiceException 业务异常
 */
@ExceptionHandler(value = ServiceException.class)
public CommonResult<?> serviceExceptionHandler(ServiceException ex) {
    log.warn("[serviceExceptionHandler]", ex);
    // 插入异常日志
    createExceptionLog(ex, null);
    return CommonResult.error(ex.getCode(), ex.getMessage());
}
```

**解读**：
- 第 5 行：`@ExceptionHandler` 处理所有 `ServiceException`
- 第 8 行：`createExceptionLog(ex, null)` 异步记录异常日志
- 第 9 行：直接用 `ex.getMessage()` 返回，未做 i18n 转换
- **ruoyi 的设计选择**：
  - **错误消息在抛出时已格式化**（如 `ServiceException(USER_NOT_EXISTS, 1L)` → "用户 1 不存在"）
  - **不依赖 `MessageSource` 二次转换**（避免 i18n 配置错误导致消息错乱）
  - **前端可独立做 i18n**（基于 `errorCode` 字典）

### 3.3 ruoyi 的多语言方案

ruoyi-vue-pro 采用 **"错误码 + 前端 i18n"** 模式：

**后端**：
- 定义 `ErrorCode` 枚举（如 `USER_NOT_EXISTS = 1001`）
- 抛异常时带错误码 + 格式化消息
- 不依赖 `MessageSource` 做服务端 i18n

**前端**：
- 维护一个 `errorCode → message` 的字典（zh-CN、en-US、ja-JP）
- 根据后端返回的 `code` 字段查表显示对应消息
- 这样后端只关注业务，前端负责多语言

**优势**：
- 后端无 i18n 配置成本
- 前端可独立翻译，无需后端修改
- 错误消息与代码解耦

## 4. 关键要点总结

- **i18n = 国际化**（多语言支持）
- **核心组件**：`MessageSource` + `LocaleResolver` + 资源文件
- **3 种 LocaleResolver**：`AcceptHeaderLocaleResolver`（API 推荐）、`CookieLocaleResolver`、`SessionLocaleResolver`
- **资源文件**：`messages_{lang}_{country}.properties`
- **使用方式**：`messageSource.getMessage(key, args, locale)`
- **ruoyi 设计**：错误码 + 前端 i18n，后端不依赖 `MessageSource`
