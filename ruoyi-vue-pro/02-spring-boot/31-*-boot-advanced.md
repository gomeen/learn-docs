## 背景

把 Web 之上的常用能力接到业务接口：校验、异步、缓存。Boot3 以对照清单为主。

## 需求

在 `/Users/xu/code/github/ruoyi-vue-pro` 中：

1. **校验**：给 demo（或现有）POST 接口的 ReqVO 加 `@NotBlank` / `@Size`，Controller 使用 `@Valid`；用错误参数触发 400/业务错误码。
2. **异步**：写 `@Async` 方法发送“假邮件”（打日志即可），调用方立即返回；确认异步线程名不同。确认项目已有异步配置（执行器 Bean）。
3. **缓存**：对某个只读查询方法加 `@Cacheable`（或使用项目封装），连续两次调用，第二次不打 SQL/远端日志（可用日志证明）。
4. **i18n（可选实现）**：找到 MessageSource / 错误码国际化用法，说明中英文切换方式。
5. **Boot3**：列出本项目当前 Spring Boot 版本，并写 3 条若升 3.x 需要改的点（javax→jakarta 等）。

## 提示

- `@Async` 同样有代理限制。
- 缓存名、Key 生成器优先复用项目现有配置。
- 校验失败是否转 CommonResult 取决于全局异常处理。

## 验收标准

- [ ] 非法参数被校验拦截，响应符合项目风格
- [ ] 异步方法在另一线程执行（日志线程名）
- [ ] 缓存命中可观察（日志或调试）
- [ ] 写出当前 Boot 版本 + 至少 3 条 3.x 迁移注意点
- [ ] 代码改动可本地启动验证
