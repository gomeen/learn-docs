# 小验证：网关与反向代理视角

> 覆盖：
- [Spring Cloud Gateway](./02-gateway.md)
>
> 预计：30～60 分钟 · 本地练习或改 ruoyi-vue-pro 仓库

## 背景

yudao-cloud 使用 Gateway；单体部署则常用 Nginx。本验证以阅读 + 路由配置为主。

## 需求

1. 阅读 Gateway 文档与（若有）cloud 仓库/模块中的路由配置。
2. 说明：鉴权、灰度、限流在网关层还是服务层更合适（结合 ruoyi 现状）。
3. 画出单体 Nginx 反代与 Cloud Gateway 两种部署拓扑差异。
4. 若环境允许，配置一条本地反代到 `yudao-server` 的路径前缀。

## 提示

- 本目录其它 Docker/CI 文档可能尚未落地，允许以现有文件+公共文档完成。
- 不要把网关密钥提交到 git。

## 验收标准

- [ ] 路由/网关配置位置明确（或说明单体不适用）
- [ ] 拓扑图完成
- [ ] 鉴权职责分层说明
- [ ] 可选：本地反代验证
- [ ] 列出 3 个上线检查项（健康检查、超时、 websocket 等）

## 延伸（选做）

- 压测网关与直连差异。
