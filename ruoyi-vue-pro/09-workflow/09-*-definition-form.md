## 背景

流程可跑的前提是：模型设计正确并成功部署，版本与分类可管理。

## 需求

1. 用 Modeler 设计/导入简单流程（至少开始→用户任务→结束）并部署。
2. 观察部署后定义版本号变化；再次修改部署，对比版本。
3. 给流程做分类，列表可筛选。
4. 在 DB 或 API 中找到流程定义/部署相关记录字段。
5. 书面说明：部署不会自动迁移已运行实例到新版本。

## 提示

- 部署后才能发起实例。
- 分类字段以项目实际模型为准。
- 本地 BPM 模块需启用。

## 验收标准

- [ ] 流程部署成功可发起
- [ ] 版本变化可观察
- [ ] 分类生效
- [ ] 定义/部署存储位置有记录
- [ ] 版本不迁移旧实例的说明正确

## 延伸（选做）

- 旧版本实例跑完，新实例走新版本。
- 导出 BPMN XML 再导入验证。
