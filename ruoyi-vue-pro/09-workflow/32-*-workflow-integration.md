## 背景

工作流要服务于业务单据状态，而不是孤立引擎。

## 需求

1. 选一个业务（请假/报销/或 demo），说明单据状态如何与流程事件同步。
2. 打开流程图高亮/追踪功能，对应后端如何计算活动节点。
3. 找到自定义任务处理器/行为扩展点。
4. 记录当前 Flowable 版本，列出与另一主版本的 2 个差异点。

## 提示

- 状态机务必幂等，监听器可能重试。
- 高亮依赖历史活动表。

## 验收标准

- [ ] 业务状态同步方案说明
- [ ] 追踪功能代码路径
- [ ] 扩展点路径
- [ ] 版本与差异记录
- [ ] 画出业务服务与 BPM 服务调用方向

## 延伸（选做）

- 业务服务 API 一键发起流程。
- 流程结束后回调更新单据。
