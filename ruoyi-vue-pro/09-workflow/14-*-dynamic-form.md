## 背景

动态表单让流程发起/审批时能采集业务字段；需搞清 schema 与提交数据存哪。

## 需求

1. 绑定或创建动态表单，发起流程时填写至少 2 个字段。
2. 在 DB/代码中找到表单 schema 与提交数据的存储位置。
3. 列出项目支持的表单组件类型 ≥5 种（对照文档/前端）。
4. 说明表单字段 key 与流程变量策略如何对齐。
5. 定位 ruoyi 表单相关 Service/Controller 入口。

## 提示

- 表单字段 key 要与流程变量策略一致。
- 注意租户与权限对表单定义的影响。
- 可与 `09-*-definition-form` 已部署流程配合。

## 验收标准

- [ ] 表单填写并随流程保存
- [ ] 存储位置说明正确
- [ ] 组件类型列表 ≥5
- [ ] 字段 key 与变量关系说明清楚
- [ ] 表单服务入口路径已记录

## 延伸（选做）

- 自定义一个表单组件。
- 对比“流程内嵌表单”与“业务单独立表”两种集成方式。
