## 背景

把后端权限与前端动态路由、按钮指令连起来。

## 需求

1. 阅读动态路由生成逻辑（后端菜单 → 前端 route）。
2. 新增一个隐藏 demo 路由页（或生成器生成页），配置菜单后仅有权限用户可见。
3. 按钮使用权限指令/组件，无权限隐藏或禁用。
4. 观察租户切换时缓存/请求头变化。
5. 画出或列出：登录后获取权限信息的主要前端文件路径。

## 提示

- 改菜单后需重新登录或刷新权限缓存。
- 权限指令名以项目为准（如 `v-hasPermi`）。
- demo 菜单 permission 标识与后端保持一致。

## 验收标准

- [ ] 动态路由链路说明清楚
- [ ] demo 菜单权限生效
- [ ] 按钮权限生效
- [ ] 租户切换影响有观察记录
- [ ] 权限相关前端文件路径列表完整

## 延伸（选做）

- 自定义一个布局内页缓存 keep-alive。
- 对比目录型菜单与菜单型菜单在路由生成上的差异。
