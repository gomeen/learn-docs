## 背景

扩展模块众多，目标是会“快速切入”：定位模块、主表、核心 API。

## 需求

任选 **两个** 模块深入（CRM / ERP / IoT / IM）：

对每个模块：
1. 找到 Maven 模块与启动依赖开关。
2. 列出 3 张核心表或 3 个核心领域对象。
3. 跑通或追踪 1 条主业务路径（如 CRM 线索→客户，ERP 采购入库，IM 发消息）。
4. 写出该模块与 system 权限/租户的集成点。

其余两个模块各用一句话说明职责。

## 提示

- 模块可按 profile 关闭，注意启用方式。
- IoT/IM 可能需额外中间件，允许代码级完成。

## 验收标准

- [ ] 两个模块的路径与主链路文档化
- [ ] 每个模块至少 3 个核心对象
- [ ] 权限/租户集成点说明
- [ ] 其余模块一句话职责
- [ ] 标明哪些仅代码阅读、哪些可运行

## 延伸（选做）

- 给所选模块补一个最小统计接口。
- 评估某模块是否适合拆独立部署。
