## 背景

真正点一次生成，把后端+前端+菜单 SQL 落到可运行。

## 需求

1. **单表**：对 demo 表生成代码，合并到工程，执行菜单 SQL，启动后完成列表增删改查。
2. **树或主子表**：任选其一，完成生成并说明与单表差异（接口、UI、表字段）。
3. 记录生成后手工修补点（权限标识、包名、模块名等）。
4. 总结一条“团队代码生成检查清单”（不少于 5 条）。

## 提示

- 生成代码冲突时优先新建表。
- 前端可能在独立仓库 yudao-ui。
- 菜单 SQL 执行后需重新登录刷权限。

## 验收标准

- [ ] 单表 CRUD 页面可完整操作
- [ ] 树或主子表完成生成并说明差异
- [ ] 手工修补点列表
- [ ] 团队检查清单 ≥ 5 条
- [ ] 测试菜单/数据可清理

## 延伸（选做）

- 主子表生成后加校验：子表至少一行。
- 把生成物改成异步导出。
