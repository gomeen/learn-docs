# 小验证：数据权限与限流

> 覆盖：
- [ruoyi 数据权限](./08-ruoyi-data-permission.md)
- [限流与防刷](./09-rate-limit.md)
>
> 预计：45～75 分钟 · 本地练习或改 ruoyi-vue-pro 仓库

## 背景

数据权限改 SQL，限流挡刷接口。两者都是安全加固的实战点。

## 需求

1. 定位数据权限拦截器与规则实现，说明“仅本人/本部门”如何落到 SQL。
2. 用两个不同数据范围的账号（若可）查询同一列表，结果集不同；或代码级推理 + 单测。
3. 给登录或 demo 接口挂限流（项目注解或 Redisson），超过阈值返回错误码。
4. 区分限流、防重放、幂等三者目标。

## 提示

- 数据权限测试账号准备较麻烦，可结合部门树数据。
- 限流 key 建议带 IP 或用户。

## 验收标准

- [ ] 数据权限入口类路径明确
- [ ] 有结果差异证据或完整代码推导
- [ ] 限流触发可复现
- [ ] 三者概念区分清楚（书面 3 行）
- [ ] 配置恢复，避免影响本地其它调试

## 延伸（选做）

- 自定义一条数据权限规则。
- 登录接口加 IP 维度限流。
