## 背景

MP 插件与自定义 SQL 是业务里的高频坑点：填充、软删、版本冲突与复杂查询。

## 需求

在 `/Users/xu/code/github/ruoyi-vue-pro` 本地库：

1. 走一次逻辑删除 → 再查应不可见；对照全局逻辑删除字段配置。
2. 观察自动填充字段：`create_time` / `updater` 等是否自动有值（库值或日志）。
3. 若表有 `version` 字段，验证乐观锁：两次读取同一 version 后先后 update，第二次应失败；否则书面说明表未启用原因。
4. 找一处 `@Select` 自定义 SQL 或 XML，说明何时该离开 Wrapper。
5. 阅读一处关联查询（联表或二次查询组装），说明项目更常用哪种。

## 提示

- 逻辑删除字段值以项目全局配置为准。
- 乐观锁需插件启用。
- 关联查询不要为了演示强行上复杂 join 污染业务表。

## 验收标准

- [ ] 逻辑删除行为正确
- [ ] 自动填充字段有证据
- [ ] 乐观锁验证完成，或书面说明未启用原因
- [ ] 自定义 SQL 场景有代码引用
- [ ] 关联查询方式有结论

## 延伸（选做）

- 试 `join` 查询的两种写法（XML vs 两次查询组装）。
- 给某 DO 增加填充字段并验证 MetaObjectHandler。
