## 背景

ruoyi 主要靠 SQL 脚本初始化；理解脚本组织方式比空谈迁移工具更重要。

## 需求

1. 浏览 `/Users/xu/code/github/ruoyi-vue-pro/sql/`，说明主库脚本如何按模块/版本组织。
2. 找一条菜单或字典的 INSERT，说明新环境如何“加一个菜单权限”。
3. 对比文档中的 Flyway 方案：当前仓库是否启用；若未启用，写出若引入应放的目录与命名规范草案。
4. 打开代码生成相关 SQL 模板/文档，说明生成器会产出哪些 SQL（菜单？建表？）。

## 提示

- 不要直接对生产执行未审核脚本。
- 菜单 SQL 常含 permission 标识，需与前端/后端一致。

## 验收标准

- [ ] 能画出 sql 目录的组织方式
- [ ] 能独立写出“新增菜单”所需的字段要点
- [ ] 明确项目当前迁移策略（脚本手工 / Flyway 等）
- [ ] 说明代码生成涉及的 SQL 产物类型
- [ ] 本地执行/dry-run 有记录（可选）

## 延伸（选做）

- 写一个可重复执行的增量脚本（存在则跳过）。
- 评估 Liquibase vs Flyway 哪个更贴合团队。
