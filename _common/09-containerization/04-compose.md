## 1. 核心概念

### 1.1 一份 Compose 文件 = 一个完整应用栈

```yaml
# 文件：docker-compose.yaml
services:        # 容器们
  web:
    image: nginx
  api:
    build: ./api
  db:
    image: postgres

networks:        # 容器网络
  default:

volumes:         # 持久化数据
  pgdata:
```

**核心要点**：
- `services` 是核心，定义每个容器
- `networks` / `volumes` 在顶层定义，供服务引用
- 默认会创建一个名为 `<项目名>_default` 的网络，所有服务互联

### 1.2 depends_on 与 healthcheck

| 字段 | 含义 |
|------|------|
| `depends_on` | 声明启动顺序（仅启动顺序，不等待服务"就绪"） |
| `condition: service_healthy` | 等待健康检查通过 |
| `healthcheck` | 定期检查服务是否就绪 |
| `restart` | 失败重启策略（`no`/`always`/`on-failure`） |

**为什么仅 `depends_on` 不够？** 数据库容器"启动"≠数据库"准备好接收连接"。必须配合 `healthcheck` 才能稳定。

### 1.3 环境变量注入：4 种方式

1. `.env` 文件：`docker-compose` 自动读取
2. `env_file` 字段：每个服务单独指定
3. `environment` 字段：内联环境变量
4. `docker run -e KEY=VALUE`：运行期覆盖

### 1.4 Profiles：按需启用

```yaml
services:
  certbot:
    profiles: ["certbot"]    # 默认不启用
```

启动：`docker compose --profile certbot up`——只有传 `--profile` 时才启动该服务。适合"按需初始化"的工具（证书申请、迁移脚本等）。

## 2. 代码示例

### 2.1 标准 Compose：WordPress + MySQL

```yaml
# 文件：docker-compose.yaml
version: '3.8'

services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_PASSWORD:-secret}
      MYSQL_DATABASE: wordpress
    volumes:
      - dbdata:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      retries: 10

  wordpress:
    image: wordpress:latest
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db
      WORDPRESS_DB_USER: root
      WORDPRESS_DB_PASSWORD: secret
    depends_on:
      db:
        condition: service_healthy
    restart: always

volumes:
  dbdata:
```

**操作**：
```bash
docker compose up -d           # 后台启动
docker compose logs -f         # 查看日志
docker compose down -v         # 停止并删除 volume
```

### 2.2 profiles 使用方式

```yaml
services:
  app:
    image: myapp
  migrate:                     # 默认不启动
    profiles: ["init"]
    image: myapp
    command: python manage.py migrate
```

```bash
# 只启动 app（不跑 migrate）
docker compose up -d

# 启动 app + 跑 migrate 一次性任务
docker compose --profile init run --rm migrate
```

### 2.3 常见错误：用 depends_on 等待就绪

```yaml
# ❌ 错误：容器启动 ≠ 服务可用
services:
  api:
    depends_on:
      - db
  db:
    image: postgres

# ✅ 正确：依赖健康检查
services:
  api:
    depends_on:
      db:
        condition: service_healthy
  db:
    image: postgres
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5
```

## 3. dify 与 ruoyi 源码解读

### 3.1 dify：15+ 服务的复杂编排

**文件位置**：`/Users/xu/code/github/dify/docker/docker-compose.yaml`
**核心代码**（行 1-15，行 1246-1260）：

```yaml
# ==================================================================
# WARNING: This file is auto-generated by generate_docker_compose
# Do not modify this file directly. Instead, update the .env.example
# or docker-compose-template.yaml and regenerate this file.
# ==================================================================

# Shared configuration using YAML anchors and env_file
x-shared-api-worker-config: &shared-api-worker-config
  env_file:
    - path: ./envs/core-services/shared.env
      required: false
    - path: ./envs/core-services/api.env
      required: false
    - path: ./envs/security.env
      required: false
    - path: ./envs/databases/db-postgres.env
      required: false
```

```yaml
networks:
  # create a network between sandbox, api and ssrf_proxy, and can not access outside.
  ssrf_proxy_network:
    driver: bridge
    internal: true
  milvus:
    driver: bridge
  opensearch-net:
    driver: bridge
    internal: true

volumes:
  oradata:
  dify_es01_data:
```

**解读**：
- 第 2-5 行：**自动生成**——dify 用 `generate_docker_compose.py` 根据 `.env.example` 渲染最终 compose，手工编辑会被覆盖
- 第 8 行：`x-shared-api-worker-config` 是 **YAML 锚点**（anchor），后续 `&shared-api-worker-config` 引用，可避免重复声明
- 第 9-67 行：把所有 `.env` 文件聚合到一处，api 和 worker 共享配置

**网络分段**（1247-1255 行）：
- `ssrf_proxy_network`（`internal: true`）：**完全不能访问外网**——SSRF 防护，容器只能通过 Squid 代理出网
- `milvus` / `opensearch-net`：独立的桥接网络，避免广播所有服务

**整体服务拓扑**：

```
       ┌────────────────────────┐
       │    ssrf_proxy (Squid)   │
       │  (出网白名单代理)       │
       └────────────────────────┘
                ↑
                │ ssrf_proxy_network (无外网)
       ┌────────┴────────────┐
   api ──┤                 ├── plugin_daemon
 worker ─┤                 ├── ssrf_proxy
       └────────┬────────────┘
                │ default
                ↓
  ┌── postgres / mysql / redis ──┐
  └── weaviate / qdrant / milvus ┘
```

### 3.2 ruoyi：极简三服务编排

**文件位置**：`/Users/xu/code/github/ruoyi-vue-pro/script/docker/docker-compose.yml`
**核心代码**（行 1-50）：

```yaml
version: "3.4"

name: yudao-system

services:
  mysql:
    container_name: yudao-mysql
    image: mysql:8
    restart: unless-stopped
    tty: true
    ports:
      - "3306:3306"
    environment:
      MYSQL_DATABASE: ${MYSQL_DATABASE:-ruoyi-vue-pro}
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD:-123456}
    volumes:
      - mysql:/var/lib/mysql/
      - ../../sql/mysql/ruoyi-vue-pro.sql:/docker-entrypoint-initdb.d/ruoyi-vue-pro.sql:ro

  redis:
    container_name: yudao-redis
    image: redis:6-alpine
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - redis:/data

  server:
    container_name: yudao-server
    build:
      context: ./yudao-server/
    image: yudao-server
    restart: unless-stopped
    ports:
      - "48080:48080"
    environment:
```

**解读**：
- 第 1 行：`version: "3.4"`——Compose 文件格式版本（新版可省略）
- 第 3 行：`name: yudao-system`——给项目一个固定名称，避免 hash 化目录名（生成 volume / network 时）
- 第 7-19 行：MySQL 通过 `${MYSQL_ROOT_PASSWORD:-123456}` 读环境变量，提供默认值
- 第 18 行：`../../sql/mysql/ruoyi-vue-pro.sql:/docker-entrypoint-initdb.d/...:ro`——**挂载 SQL 初始化脚本**，MySQL 首次启动会自动执行
- 第 22-27 行：Redis 用 `6-alpine` 精简镜像，数据挂载到 `redis` volume
- 第 29-32 行：`server` 通过 `build: ./yudao-server/` 触发 Dockerfile 构建

**对比**：
- **dify**：15+ 服务 + 多 network + 多 volume + YAML 锚点 + 自动生成——专为"开箱即用"的多产品部署设计
- **ruoyi**：3 服务（DB / Cache / App）+ 1 network + 1 volume pair——专为"本地开发"极简化设计
- **建议**：生产参照 dify 模式（拆分外部数据库、健康检查、Profile），开发参照 ruoyi 模式（一键拉起）

## 4. 关键要点总结

- **services / networks / volumes** 三大核心对象
- **`depends_on` + `condition: service_healthy`** 才能等到服务"可用"
- **`profiles`** 用于按需初始化工具（迁移脚本、证书申请）
- **环境变量注入优先级**：`.env` < `env_file` < `environment` < `docker run -e`
- **网络分段**：用 `internal: true` 切断外网（SSRF 防护）
- **YAML 锚点 (`&`/`*`)** 在多服务共享配置时减少重复
