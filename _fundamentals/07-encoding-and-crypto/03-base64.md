# 1.3 Base64 编码

> Base64 是最常见的二进制到文本编码方案。JWT、URL 参数、邮件附件都用 Base64。

## 1. 核心概念

### 1.1 Base64 的原理

```
3 字节 = 24 位 → 分成 4 个 6 位 → 每个 6 位映射到 1 个字符
                  ↑
              64 个可打印字符（A-Z, a-z, 0-9, +, /）
```

### 1.2 字符集

```
标准 Base64: ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
URL 安全:     ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789-_
```

### 1.3 Base64 变体

| 变体 | 用途 |
|------|------|
| 标准 Base64 | 通用 |
| URL Safe Base64 | URL 参数（把 `+`/`/` 换成 `-`/`_`） |
| MIME Base64 | 每 76 字符换行 |

### 1.4 Base64 不是加密！

**Base64 是编码，不是加密**：
- 不需要密钥，任何人都能解码
- 用途：让二进制数据可作为文本传输
- 不能提供机密性

## 2. 代码示例

### 2.1 Python Base64 基础

```python
import base64

# 编码
text = "Hello, World!"
encoded = base64.b64encode(text.encode("utf-8"))
print(encoded)  # b'SGVsbG8sIFdvcmxkIQ=='

# 解码
decoded = base64.b64decode(encoded).decode("utf-8")
print(decoded)  # Hello, World!
```

### 2.2 URL Safe Base64

```python
import base64

data = b"subjects?"  # 含 ? 字符

# ❌ 标准 Base64 有 +/=
standard = base64.b64encode(data)
print(standard)  # b'c3ViamVjdHM/'  → '/' 在 URL 中需转义

# ✅ URL Safe（用 -_ 替代 +/）
url_safe = base64.urlsafe_b64encode(data)
print(url_safe)  # b'c3ViamVjdHM_'

# URL Safe 解码
decoded = base64.urlsafe_b64decode(url_safe)
```

### 2.3 二进制数据编码

```python
import base64

# 图像 → Base64（用于内嵌到 HTML）
with open("logo.png", "rb") as f:
    image_bytes = f.read()
b64_image = base64.b64encode(image_bytes).decode("utf-8")

# HTML 内嵌图像
html = f'<img src="data:image/png;base64,{b64_image}" />'

# Base64 还原图像
image_data = base64.b64decode(b64_image)
with open("logo_copy.png", "wb") as f:
    f.write(image_data)
```

### 2.4 手写 Base64（理解原理）

```python
CHARS = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/"

def manual_base64_encode(data: bytes) -> str:
    """手动实现 Base64 编码（教学用）"""
    result = []
    for i in range(0, len(data), 3):
        chunk = data[i:i+3]
        # 不足 3 字节，补 0
        while len(chunk) < 3:
            chunk += b"\x00"

        # 3 字节 → 4 个 6 位
        n = (chunk[0] << 16) | (chunk[1] << 8) | chunk[2]
        chars = [
            CHARS[(n >> 18) & 0x3F],
            CHARS[(n >> 12) & 0x3F],
            CHARS[(n >> 6) & 0x3F],
            CHARS[n & 0x3F],
        ]

        # 处理填充
        if len(data[i:i+3]) < 3:
            chars[-(3 - len(data[i:i+3])):] = ["="] * (3 - len(data[i:i+3]))

        result.extend(chars)
    return "".join(result)


# 测试
print(manual_base64_encode(b"Man"))  # TWFu
```

## 3. dify / ruoyi 仓库源码解读

### 3.1 dify 密码存储用 Base64

**位置**：`/Users/xu/code/github/dify/api/services/account_service.py`
**核心代码**（行 380-390）：

```python
import base64
from libs.password import hash_password

def setup_account_password(account, password: str, salt: bytes):
    """设置账户密码"""
    # 1. 哈希密码
    password_hashed = hash_password(password, salt)

    # 2. Base64 编码（存入数据库）
    base64_password_hashed = base64.b64encode(password_hashed).decode()
    account.password = base64_password_hashed
```

**解读**：
- 哈希结果是字节 → 用 Base64 转字符串存入数据库
- Base64 让二进制数据可以作为 VARCHAR 存储

### 3.2 JWT 用 Base64（典型应用）

**位置**：`/Users/xu/code/github/ruoyi-vue-pro/yudao-framework/yudao-spring-boot-starter-security/`
**核心代码**：

```java
// JWT 头部 + Payload 用 Base64 URL Safe 编码
String header = Base64.getUrlEncoder().encodeToString("{\"alg\":\"HS256\"}".getBytes());
String payload = Base64.getUrlEncoder().encodeToString("{\"sub\":\"user-1\"}".getBytes());
String signature = sign(header + "." + payload, secret);
// JWT: header.payload.signature
```

**解读**：
- JWT 用 Base64 URL Safe（避免 URL 转义问题）
- **JWT = Base64(header) + "." + Base64(payload) + "." + signature**

## 4. 关键要点总结

- Base64 是二进制到文本的编码（不是加密）
- 3 字节 → 4 字符；4 字符 → 3 字节
- URL Safe 用 `-_` 替代 `+/`，避免 URL 转义
- JWT 用 Base64 URL Safe
- dify 用 Base64 存哈希密码的字节形式
